## The Learning Path

Here is the bird's-eye view of your journey. Don't worry about the jargon yet; we’ll break it down piece by piece.

## 1. The Core Fundamentals (The "Big Three")

### HTML (HyperText Markup Language)

The skeleton of every website. You'll learn how to structure content using elements.

* **Focus on:** Semantic HTML (using `<main>`, `<nav>`, `<footer>`) and accessibility.

### CSS (Cascading Style Sheets)

The skin and clothes. This is how you add colors, fonts, and layouts.

* **Focus on:** Flexbox, CSS Grid, and Responsive Design (making sites look good on phones).

### JavaScript (The Logic)

The brain of the site. It makes things interactive—like dropdown menus, dark mode toggles, and data fetching.

* **Focus on:** ES6+ syntax, DOM manipulation, and Async/Await.

## 2. Version Control & Collaboration

### Git & GitHub

Git is your "Undo" button, and GitHub is your "Cloud Storage + Portfolio."

* **Git:** Learn how to track changes (commits) and experiment with new features (branches).
* **GitHub:** Learn how to push your code online and collaborate with others through Pull Requests.

## 3. The Modern Ecosystem

### npm (Node Package Manager)

The world's largest software registry. You don't need to reinvent the wheel—you can use "packages" built by others to speed up your development.

### React

The industry-standard library for building user interfaces. It allows you to build "Components" (reusable UI blocks like buttons or cards).

* **Key Concept:** In 2026, focus on **Server Components** and the latest **Hooks**.

### Tailwind CSS

A utility-first CSS framework. Instead of writing long CSS files, you apply styles directly in your HTML/JSX classes. It makes styling incredibly fast and consistent.

## 4. Quality & Testing

### Vitest

Building it isn't enough; you need to make sure it doesn't break. **Vitest** is a blazing-fast unit testing framework.

* **Why?** It ensures that when you add a new feature, your old ones still work.

:::tip Success Tip
Don't try to learn everything at once. Spend **80% of your time coding** and **20% watching tutorials**. Building a simple, ugly project is better than watching a perfect video.
:::
